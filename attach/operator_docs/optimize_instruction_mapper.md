# 优化Instruction (optimize_instruction_mapper)

## 算子功能

这是一个智能指令扩展工具,它能把简单的问题变成详细的、结构化的指令,但不回答问题本身,只让问题变得更具体、更全面。


## 处理逻辑

### 第一步:验证参数

检查必填参数:
- model_url不能为空
- auth_token不能为空
- 参数无效会抛出错误

### 第二步:构建请求

准备API请求:
- 使用固定的系统提示词
- 将原文本作为用户消息
- 构建OpenAI格式的请求

### 第三步:调用API

发送请求到AI模型:
- POST请求到指定API
- 使用Bearer Token认证
- 60秒超时
- 非流式响应

### 第四步:获取结果

处理API响应:
- 从响应中提取优化后的指令
- 直接替换原文本
- 不保留原文本

### 第五步:错误处理

处理各种异常:
- HTTP错误:保留原文本
- 解析错误:保留原文本
- 其他异常:保留原文本
- 记录错误日志


### 示例

**配置:**
```yaml
模型地址: "https://api.deepseek.com/chat/completions"
模型名称: "deepseek-chat"
授权令牌: "sk-xxxxxxxxxxxxx"
```

**输入数据:**
```json
{
  "text": "今天天气怎么样?"
}
```


**输出数据:**
```json
{
  "text": "请提供今天的详细天气信息,包括:1)当前温度和体感温度;2)天气状况(晴天、多云、雨天等);3)湿度和风力情况;4)空气质量指数(AQI);5)未来3-6小时的天气变化趋势;6)适合的穿衣建议和出行提示。"
}
```
