# 特殊后缀过滤 (suffix_filter)

## 算子功能

这是一个用来筛选文件的工具，它会检查文件的后缀名（扩展名），然后根据后缀名是否在允许列表中来决定是保留还是丢弃这个文件。(只支持单个扩展名过滤且需要有`__dj__suffix__`字段,扩展名只能是小写)

## 处理逻辑

### 第一步：获取文件后缀

从数据中读取 `__dj__suffix__` 字段的值

### 第二步：检查是否在允许列表中

- 如果设置了允许列表：
  - 后缀在列表中 → 保留
  - 后缀不在列表中 → 丢弃
- 如果没有设置允许列表：
  - 保留所有文件

### 示例

**配置：**
```yaml
后缀: .txt
```

**输入数据：**
```json
[
  {"text": "文本文件", "__dj__suffix__": ".txt"},
  {"text": "Markdown文件", "__dj__suffix__": ".md"},
  {"text": "PDF文档", "__dj__suffix__": ".pdf"},
  {"text": "Word文档", "__dj__suffix__": ".docx"},
  {"text": "图片文件", "__dj__suffix__": ".jpg"},
  {"text": "代码文件", "__dj__suffix__": ".py"}
]
```

**输出数据：**
```json
[
  {"text": "文本文件", "__dj__suffix__": ".txt"}
]
```
